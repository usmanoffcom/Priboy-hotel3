# Проверка скорости и CDN — priboy1.ru

## Текущая конфигурация (по коду)

| Что | Статус |
|-----|--------|
| Оптимизация изображений Next.js (`formats`, `deviceSizes`, `minimumCacheTTL`) | ✅ В `next.config.mjs` |
| `remotePatterns` для внешних картинок | ✅ Например `framerusercontent.com` |
| Заголовки кеша для `/_next/static` и расширений статики | ✅ В `next.config.mjs` |
| **`assetPrefix` (внешний CDN)** | По умолчанию **пусто** — статика с того же origin, что и сайт. Включение только через **`NEXT_PUBLIC_CDN_URL`** в окружении сборки. |

В `next.config.mjs` указано, что отдельный Selectel-URL раньше давал CORS/404, поэтому префикс CDN не задаётся в репозитории жёстко.

---

## CDN

1. **Без переменной:** `assetPrefix: ''` — всё с `priboy1.ru` (или с того хоста, где крутится Next).
2. **С переменной:** задать при `pnpm build` / на CI, например `NEXT_PUBLIC_CDN_URL=https://…` — тогда статика `_next/static` пойдёт с указанного базового URL (нужны корректные CORS и отдача файлов на CDN).
3. **Nginx на VDS:** в `deploy.sh` уже есть `proxy_cache_valid` и Cache-Control для статики через прокси.

Дополнительно: **Cloudflare** (или другой reverse CDN) перед доменом — по желанию, см. чек-лист ниже.
